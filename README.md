# بات تلگرام تبدیل فایل به لینک مستقیم (چند هاستی)

این بات هر فایلی که برایش بفرستید را همزمان در چند هاست مختلف آپلود می‌کند و لینک دانلود مستقیم هرکدام را برمی‌گرداند.

## سرویس‌های فعال

| سرویس       | نیاز به API Key؟ | توضیح |
|-------------|-------------------|-------|
| Pixeldrain  | ❌ اختیاری        | بدون کلید هم کار می‌کند (آپلود مهمان). با `PIXELDRAIN_API_KEY` فایل به حساب شما متصل می‌شود. |
| GoFile      | ❌ اختیاری        | بدون کلید هم کار می‌کند (آپلود مهمان). با `GOFILE_API_TOKEN` فایل به حساب گوفایل شما متصل می‌شود. |
| storage.to  | ❌ هیچ‌وقت لازم نیست | کاملاً بدون حساب و بدون کلید کار می‌کند؛ به همین خاطر هیچ متغیری برایش تعریف نشده. |

هر سه سرویس **بدون نیاز به API Key** کار می‌کنند. کلیدهای بالا کاملاً اختیاری‌اند و فقط در صورت تمایل به اتصال آپلودها به حساب شخصی استفاده می‌شوند.

## راه‌اندازی

۱. فایل `.env.example` را کپی کرده و به `.env` تغییر نام دهید:

```bash
cp .env.example .env
```

۲. مقادیر زیر را در `.env` پر کنید:

- `TELEGRAM_BOT_TOKEN` — از [@BotFather](https://t.me/BotFather)
- `TELEGRAM_API_ID` و `TELEGRAM_API_HASH` — از https://my.telegram.org (برای اجرای سرور لوکال Bot API لازم است تا فایل‌های تا ۲ گیگابایت پشتیبانی شوند)
- (اختیاری) `PIXELDRAIN_API_KEY`
- (اختیاری) `GOFILE_API_TOKEN`

۳. ساخت و اجرای ایمیج داکر:

```bash
docker build -t telegram-multi-uploader .
docker run --env-file .env -p 8081:8081 telegram-multi-uploader
```

## نحوه‌ی کارکرد سیستم پیشرفت (Progress)

- **مرحله‌ی دریافت از تلگرام:** سرور لوکال Bot API فایل را مستقیم روی دیسک می‌نویسد؛ بات با پایش حجم فایل روی دیسک یک نوار پیشرفت واقعی نشان می‌دهد (دیگر پیام ثابت نمی‌ماند).
- **مرحله‌ی آپلود به هر هاست:** هرکدام از سرویس‌ها (Pixeldrain، GoFile، storage.to) به‌صورت stream-based آپلود می‌شوند و درصد، حجم، سرعت و زمان باقیمانده هر چند ثانیه به‌روزرسانی می‌شود.
- در پایان، اگر آپلود در یک یا چند سرویس ناموفق باشد، همچنان لینک‌های موفق نمایش داده می‌شوند و خطای سرویس‌های ناموفق به‌صورت جدا گزارش می‌شود.

## افزودن/حذف سرویس‌ها

هر سرویس آپلود یک فایل جدا در `src/uploaders/` دارد و رابط مشترک `UploaderService` (در `src/types.ts`) را پیاده می‌کند. برای افزودن سرویس جدید کافیست فایل مشابهی بسازید و آن را به آرایه‌ی `services` در `src/index.ts` اضافه کنید.
